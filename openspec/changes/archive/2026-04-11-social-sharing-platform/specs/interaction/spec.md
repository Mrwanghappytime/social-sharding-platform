## 新增需求

### 需求:点赞动态
系统必须支持登录用户对动态进行点赞，每个用户对同一动态只能点赞一次。

#### 场景:成功点赞
- **当** 登录用户对未点赞的动态进行点赞
- **那么** 系统创建点赞记录并返回成功

#### 场景:重复点赞
- **当** 用户对已点赞的动态再次点赞
- **那么** 系统返回错误码 409 Conflict

#### 场景:点赞自己的动态
- **当** 用户对自己发布的动态进行点赞
- **那么** 系统允许点赞

### 需求:取消点赞
系统必须支持登录用户取消对动态的点赞。

#### 场景:成功取消点赞
- **当** 登录用户对已点赞的动态取消点赞
- **那么** 系统删除点赞记录并返回成功

#### 场景:取消未点赞的动态
- **当** 用户对未点赞的动态取消点赞
- **那么** 系统返回错误码 404 Not Found

### 需求:评论动态
系统必须支持登录用户对动态发表评论。

#### 场景:成功评论
- **当** 登录用户对动态提交评论内容
- **那么** 系统创建评论并返回评论信息

#### 场景:评论内容为空
- **当** 用户提交空评论内容
- **那么** 系统返回错误码 400 Bad Request

### 需求:删除评论
系统必须支持评论发布者删除自己的评论，或动态发布者删除任意评论。

#### 场景:发布者删除自己评论
- **当** 评论发布者请求删除自己的评论
- **那么** 系统删除评论并返回成功

#### 场景:动态发布者删除他人评论
- **当** 动态发布者请求删除该动态下的任意评论
- **那么** 系统删除评论并返回成功

#### 场景:非相关方删除评论
- **当** 非评论发布者且非动态发布者的用户请求删除评论
- **那么** 系统返回错误码 403 Forbidden

### 需求:获取动态评论列表
系统必须支持获取指定动态的评论列表，按创建时间正序排列。

#### 场景:获取评论列表
- **当** 请求指定动态的评论列表
- **那么** 系统返回该动态的评论列表

### 需求:获取动态点赞状态和计数
系统必须支持获取动态的点赞数和当前用户的点赞状态。

#### 场景:获取点赞信息
- **当** 请求动态的点赞信息
- **那么** 系统返回点赞数和当前用户是否已点赞
