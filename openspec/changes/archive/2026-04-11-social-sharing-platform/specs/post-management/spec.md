## 新增需求

### 需求:发布动态
系统必须支持用户发布动态，包含标题、内容和媒体（图片或视频，不能混搭）。图片最多9张，视频最多1个且最大50MB。

#### 场景:成功发布纯文字动态
- **当** 用户提交不含媒体的动态（标题和内容）
- **那么** 系统创建动态并返回动态信息

#### 场景:成功发布带图片动态
- **当** 用户提交带1-9张图片的动态
- **那么** 系统创建动态并关联图片，返回动态信息

#### 场景:成功发布带视频动态
- **当** 用户提交带单个视频（≤50MB）的动态
- **那么** 系统创建动态并关联视频，返回动态信息

#### 场景:图片与视频混搭
- **当** 用户尝试同时提交图片和视频
- **那么** 系统返回错误码 400 Bad Request

#### 场景:图片超过9张
- **当** 用户提交超过9张图片的动态
- **那么** 系统返回错误码 400 Bad Request

#### 场景:视频超过50MB
- **当** 用户提交超过50MB的视频
- **那么** 系统返回错误码 400 Bad Request

### 需求:获取动态详情
系统必须支持通过动态 ID 获取动态详情，包括发布者信息、媒体列表、点赞数和评论数。

#### 场景:动态存在
- **当** 请求一个存在的动态 ID
- **那么** 系统返回动态详情

#### 场景:动态不存在
- **当** 请求一个不存在的动态 ID
- **那么** 系统返回错误码 404 Not Found

### 需求:删除动态
系统必须支持动态发布者删除自己的动态。

#### 场景:发布者删除动态
- **当** 动态发布者请求删除该动态
- **那么** 系统删除动态及其关联的媒体文件

#### 场景:非发布者删除动态
- **当** 非发布者请求删除他人动态
- **那么** 系统返回错误码 403 Forbidden

### 需求:获取用户动态列表
系统必须支持获取指定用户的动态列表，按创建时间倒序排列。

#### 场景:获取用户动态
- **当** 请求指定用户的动态列表
- **那么** 系统返回该用户的动态列表

#### 场景:用户无动态
- **当** 请求一个没有发布过动态的用户
- **那么** 系统返回空列表

### 需求:获取首页动态流
系统必须支持获取所有用户的动态列表（按时间倒序），分页返回。

#### 场景:获取首页动态
- **当** 请求首页动态列表
- **那么** 系统返回按时间倒序的动态列表

#### 场景:分页
- **当** 请求首页动态列表并指定页码和每页数量
- **那么** 系统返回对应分页的动态列表
