## 新增需求

### 需求:上传图片
系统必须支持上传图片文件，返回可访问的 URL。

#### 场景:成功上传单张图片
- **当** 用户上传单个图片文件（支持的格式：jpg、jpeg、png、gif、webp）
- **那么** 系统保存文件并返回 URL

#### 场景:成功上传多张图片
- **当** 用户上传多个图片文件（最多9张）
- **那么** 系统保存所有文件并返回 URL 列表

#### 场景:不支持的图片格式
- **当** 用户上传不支持格式的文件
- **那么** 系统返回错误码 400 Bad Request

#### 场景:图片数量超过9张
- **当** 用户尝试上传超过9张图片
- **那么** 系统返回错误码 400 Bad Request

### 需求:上传视频
系统必须支持上传视频文件，最大50MB。

#### 场景:成功上传视频
- **当** 用户上传视频文件（支持的格式：mp4、mov、avi、webm），大小 ≤ 50MB
- **那么** 系统保存文件并返回 URL

#### 场景:视频超过50MB
- **当** 用户上传超过50MB的视频
- **那么** 系统返回错误码 400 Bad Request

#### 场景:不支持的视频格式
- **当** 用户上传不支持格式的视频
- **那么** 系统返回错误码 400 Bad Request

### 需求:访问上传的文件
系统必须支持通过返回的 URL 直接访问上传的文件。

#### 场景:访问图片
- **当** 用户请求图片 URL
- **那么** 系统返回图片文件

#### 场景:访问视频
- **当** 用户请求视频 URL
- **那么** 系统返回视频文件

#### 场景:文件不存在
- **当** 用户请求一个不存在的文件
- **那么** 系统返回错误码 404 Not Found

### 需求:删除文件
系统必须支持删除与指定动态关联的文件。

#### 场景:删除动态关联的文件
- **当** 动态被删除时
- **那么** 系统删除该动态关联的所有媒体文件
