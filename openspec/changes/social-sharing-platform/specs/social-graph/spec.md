## 新增需求

### 需求:关注用户
系统必须支持登录用户关注其他用户，不能重复关注自己。

#### 场景:成功关注
- **当** 登录用户关注一个未关注的用户
- **那么** 系统创建关注记录并返回成功

#### 场景:关注自己
- **当** 用户尝试关注自己
- **那么** 系统返回错误码 400 Bad Request

#### 场景:重复关注
- **当** 用户尝试关注已关注的用户
- **那么** 系统返回错误码 409 Conflict

### 需求:取消关注
系统必须支持登录用户取消关注已关注的用户。

#### 场景:成功取消关注
- **当** 登录用户取消关注一个已关注的用户
- **那么** 系统删除关注记录并返回成功

#### 场景:取消未关注用户
- **当** 用户尝试取消关注一个未关注的用户
- **那么** 系统返回错误码 404 Not Found

### 需求:获取用户关注列表
系统必须支持获取指定用户的关注列表，按关注时间倒序排列。关注列表按 following_id % 16 分表存储。

#### 场景:获取关注列表
- **当** 请求指定用户的关注列表
- **那么** 系统返回该用户的关注列表，包含用户基本信息

#### 场景:用户未关注任何人
- **当** 请求一个没有关注任何人的用户关注列表
- **那么** 系统返回空列表

### 需求:获取用户粉丝列表
系统必须支持获取指定用户的粉丝列表，按关注时间倒序排列。粉丝列表按 follower_id % 16 分表存储。

#### 场景:获取粉丝列表
- **当** 请求指定用户的粉丝列表
- **那么** 系统返回该用户的粉丝列表，包含用户基本信息

#### 场景:用户无粉丝
- **当** 请求一个没有粉丝的用户粉丝列表
- **那么** 系统返回空列表

### 需求:获取关注数和粉丝数
系统必须支持获取指定用户的关注数和粉丝数。

#### 场景:获取计数
- **当** 请求指定用户的关注数和粉丝数
- **那么** 系统返回关注数和粉丝数

### 需求:判断关注状态
系统必须支持当前登录用户判断是否关注了指定用户。

#### 场景:已关注
- **当** 当前用户已关注指定用户
- **那么** 系统返回已关注状态

#### 场景:未关注
- **当** 当前用户未关注指定用户
- **那么** 系统返回未关注状态
