## 新增需求

### 需求:用户注册
系统必须支持用户通过用户名和密码进行注册。用户名必须唯一。密码必须加密存储（BCrypt）。

#### 场景:成功注册
- **当** 用户提交有效的用户名和密码
- **那么** 系统创建新用户并返回用户信息（不含密码）

#### 场景:用户名已存在
- **当** 用户提交一个已存在的用户名
- **那么** 系统返回错误码 409 Conflict

### 需求:用户登录
系统必须支持用户通过用户名和密码登录，成功后返回 JWT Token。

#### 场景:成功登录
- **当** 用户提交正确的用户名和密码
- **那么** 系统返回有效的 JWT Token

#### 场景:凭据错误
- **当** 用户提交错误的密码
- **那么** 系统返回错误码 401 Unauthorized

### 需求:获取当前用户信息
系统必须支持通过 JWT Token 获取当前登录用户的详细信息。

#### 场景:Token 有效
- **当** 用户携带有效 JWT 请求用户信息
- **那么** 系统返回用户详细信息

#### 场景:Token 无效或过期
- **当** 用户携带无效或过期的 JWT 请求用户信息
- **那么** 系统返回错误码 401 Unauthorized

### 需求:更新用户头像
系统必须支持用户更新自己的头像 URL。

#### 场景:成功更新头像
- **当** 用户上传头像并提交新的头像 URL
- **那么** 系统更新用户头像并返回更新后的用户信息

#### 场景:用户未登录
- **当** 未登录用户尝试更新头像
- **那么** 系统返回错误码 401 Unauthorized
