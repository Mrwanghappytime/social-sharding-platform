优势对比：

❌ 旧方案（per-conversation channel）：N 个会话 = N 个 Redis channel，用户进出会话动态订阅/取消订阅，多实例场景消息可能丢失。

✅ 新方案（全局单 channel）：只有 1 个 Redis channel，实例启动时订阅一次，消息全量广播 + 本地路由过滤，多实例天然支持水平扩展。

核心功能特性

实时私聊

基于 WebSocket 的实时消息传输，支持单用户单连接复用，进入会话时 JOIN，离开时 LEAVE，空闲 5 分钟自动关闭连接。

消息类型

支持文本消息和图片消息，图片默认压缩（1280px 最大边长，JPEG 质量 0.8），可选择发送原图。

会话管理

会话使用唯一键 (user1_id, user2_id) 并规范化为 user1_id < user2_id，确保每个对话只创建一个会话记录。

通知系统

私聊通知与现有通知系统整合，每个会话对应一个 MESSAGE 类型通知，显示未读消息数和最后消息预览。

Redis 广播模型

使用单一全局 channel（message:push），实例启动时订阅一次；消息 publish 时携带完整 MessageDTO，各实例本地路由到 receiver 所在会话，避免 channel 爆炸和多实例消息丢失。

水平扩展友好

message-service 可任意扩容，用户 WebSocket 可连接到任一实例；发送端与接收端不需要在同一实例，天然支持集群部署。

技术栈

Vue 3 Pinia Element Plus TypeScript Spring Boot 3.2 Dubbo 3.2 Spring Cloud Gateway Nacos MySQL Redis WebSocket Docker

数据库设计

conversations 表：存储对话信息，包含 id、user1_id、user2_id（规范化为 user1 < user2）、last_message_preview、last_message_time、unread_count 等字段。

messages 表：存储消息内容，包含 id、conversation_id、sender_id、receiver_id、content、type（TEXT/IMAGE）、image_url、original_image_url、created_at 等字段。
